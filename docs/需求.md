## 这个demo最后要做到上线

最短可上线的 MVP 功能清单
### P0

账号体系：短信/邮箱（先邮箱也行）或校内统一认证（以后再接）

帖子：发、删（软删）、列表、详情

评论：发、删（软删）、楼中楼（parent_id）

访问控制：游客可看 / 登录可发

基础反滥用：发帖/评论限流（按 IP + 用户）

举报入口 + 管理员处理（最简后台也行）

日志：每个请求记录 userId、ip、耗时、状态码
 
### P1（强烈建议）

图片上传（对象存储）

搜索（先标题/正文简单 like，再上全文索引）

置顶/热门（运营抓手）

站内通知（被评论/被回复）

### P2（聊天室）

先做“帖子下实时讨论”而不是“全站聊天室”

WebSocket 会话鉴权 + 限流 + 内容过滤

消息持久化策略（最开始可以只存最近 N 条）
